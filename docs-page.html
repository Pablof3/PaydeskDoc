Introducción Ultima Actulizacion: 2019-07-16

Este documento describe los requerimientos de software de Paydesk, cuyo objetivo principal es descentralizar las colas de distintas areas de la universidad privada Domingo Savio con un sistema informático.

Instalación

El modo de acceso al sistema se realiza mediante una direccion url, que le sera provisto por el administrador, conjuntamente con unas creedenciales para poder acceder al mismo, estos estaran relacionados a un tipo de usuario sobre el sistema, con el cual cumplira un rol en el mismo .
El formato de url para el acceso al sistema puede ser parecido al siguiente:

http://localhost/paydesk2.0/public/

Emisor de Fichas

Para el acceso a la vista de emision de fichas se debe acceder al apartado de tickets /tickets, esto le desplegara la vista para que un usuario pueda realizar la peticion de una nueva ficha de atencion.

La url se vera aproximadamente de la siguiente manera:

http://localhost/paydesk2.0/public/tickets

Pantalla de orden de atencion

La vista de orden de atencion, se encuentra en el apartado /turno ,el cual muestra la asignaciones de fichas a ventanillas, tambien cuenta con una seccion para videos institucionales, promocionales configurables, comunicados importantes y anuncios.

La url se asemejara al siguiente ejemplo:

http://www.Paydesk.com/turno

Inicio de sesion

EL sistema esta protegido por credenciales de nombre de usuario y contraseñas para que personas externas a cuentas diferentes puedan corremper la cuenta de otros.

Iniciar sesión

El inicio de sesion en un principio tiene una cuenta por defecto de administrador con el nombre de usuario "Administrador" y conraseña "administrador123", con estos credenciales correctamente escritos podremos iniciar sesion, ademas tambien podemos recordar el nombre de usuario y la contraseña marcando la casilla recordarme.

En caso de que los credenciales esten mal escritos el sistema no permitira el inicio de sesion y mostrará error.

En el ejemplo se cometerá un error al ingresar el nombre de usuario y nos mostrara un error y luego recien se ingresará los credenciales correctos el administrador ficticio se llama Cosme Fulanito, una vez ingresado al sistema se mostrara el cierre de sesion.

Este procedimiento se aplica para cualquier tipo de usuario siempre y cuando el administrador los haya creado con sus correspondientes nombres de usuario y contraseñas

Pagina de inicio

Una vez que se valida el inicio de sesión el sistema mostrará la pagina de inicio.

Estado de ventanillas

El estado de ventanillas muestra solo al administrador aquellas ventanilas como cajas o personal de plataforma que esta activos de color verde y los que no de color rojo.

Menu

En el lado izquierdo de la aplicación muestra un menu en la que el adminisyrador puede gestionar a los usuarios, las ventanillas, las colas, areas, videos, comunicados, anuncios y la administracion de la base de datos de los estudiantes.

Estadisticas

Las estadisticas contemplan la atencion de fichas y los tiempos promedios de atencion por ventanillas, areas, general y por usuario, esto cambia dinamicamente segun el año, el mes y el dia.
Las estadisticas de atencion ademas contemplan cuantas fichas fueron atendidas o aceptadas y cuantas fichas fueron canceladas.

Usuarios

Los usuarios registrados en el sistema son los unicos que pueden ejercer acciones sobre este.

Listar Usuarios

La lista de usuarios muestra todos las personas registradas por el administrador con su respectivos nombres, apellidos telefenos y el rol que desempeña, el usuario registrado puede ser gestionado por el administrador como ser ver la informacion, actualizar sus datos, eliminar del sistema y ademas de crear uno nuevo.

Crear usuario administrador

Al crear un usuario administrador como el que esta por defecto quiere decir que este tendra control total del sistema.

Crear usuario

Cuando se crea un usuario convencional sin privelegios de administrador este tiene roles que cumplir como ser cajero, atencion al cliente, gestionar los videos y comunicados estos no podran crear otros usuarios ni generar acciones que el administrador posee.
